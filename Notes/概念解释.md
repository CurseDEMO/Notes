# Git
- 一种版本控制系统
- 比如你写了一个文档, 写了3次, 第一次是初稿, 第二次是修改稿,第三次是终稿. 有一天你发现你需要回到某一次的稿子, 但摆在你面前有一个问题: 怎样获取那一次的稿子? 首先你可以创建三个不同文件, 分别是初稿修改稿和终稿, 或者分三个文件夹. 不过倒霉的是如果你选择三次都写在同一个文件上, 那很糟糕了,你得自己回忆了.
- 于是, Git诞生了, 它可以保存每一个版本的快照, 即保留每一个版本, 方便你随时回溯.
- 这就是Git的主要功能
# GitHub
- Git本身运作在本地计算机上.
- 但数据有可能丢失, 于是就有远程托管仓库GitHub, 用来存放你的文件及其快照.
- 本地的Git可以从远程仓库获取相关资料(提交)
# Git中的工作区域
## Git仓库
- 存放所有数据(不是仅仅是工作目录的, 还包括过去的快照(具有时光机的能力))
## 工作目录
- 某个版本的数据会存放在这里,供你操作
## 暂存区域
- ⼀个文件，保存了下次将提交的文件列表信息(下次提交什么到Git仓库)
## 三者关系
### 流程
1. 在⼯作⽬录中修改⽂件。 
2. 暂存⽂件，将⽂件的快照放⼊暂存区域。 
3. 提交更新，找到暂存区域的⽂件，将快照永久性存储到 Git 仓库⽬录。

# Git中的文件状态
![](https://cdn.jsdelivr.net/gh/CurseDEMO/image-bed@main/img/20260617100044170.png)
## 已跟踪(tracked)
- 已被纳入版本控制系统的文件
### 已提交(committed)
- 文件在Git仓库
### 已修改(modified)
- 文件和在Git仓库中的文件不一样且在工作目录但不在暂存区域
### 未修改(unmodified)
- 文件和在Git仓库中的文件一致
### 已暂存(staged)
- 文件已放入暂存区域
## 未跟踪(untracked)
- 未被纳入版本控制系统的文件
